🔍 Phase 0: Foundation & Assessment

0.1 Discovery & Dependency Mapping

Before touching Azure, you must understand what you have. Use Azure Migrate (with appliance-based discovery) to perform granular dependency mapping.

# Install Azure Migrate appliance (VMware/Hyper-V/Physical) # Download from Azure Portal > Azure Migrate > Discover > Download OVA/VHD # After discovery, export dependency data $Assessment = Get-AzMigrateAssessment ` -ProjectName "DC-Migration-Project" ` -ResourceGroupName "rg-migration" ` -AssessmentName "Production-Assessment" # Key outputs: CPU/Memory utilization, disk IOPS, network dependency graph
Capture East-West traffic patterns, North-South volumes, latency requirements per tier, bandwidth baselines (95th percentile), and IP addressing schemes to avoid RFC1918 overlaps.

1.2 Network Topology: Hub-and-Spoke Model

The enterprise standard for Azure network architecture.

Hub VNet (Shared Services)

# Address Space: 10.0.0.0/16 (65,536 IPs) # Subnets: GatewaySubnet: 10.0.0.0/27 # ExpressRoute/VPN gateways AzureFirewallSubnet: 10.0.0.32/26 # Azure Firewall AzureBastionSubnet: 10.0.0.96/26 # Bastion for secure RDP/SSH AppGatewaySubnet: 10.0.0.160/27 # Application Gateway WAF ADDS-Subnet: 10.0.1.0/24 # Domain Controllers SharedServices: 10.0.2.0/24 # DNS, NTP, monitoring

Spoke VNet (Application Workloads)

# Address Space: 10.1.0.0/16 (Production) # Address Space: 10.2.0.0/16 (Non-Production) # Subnets per Spoke: WebTier: 10.1.0.0/24 # NSG: Allow 80/443 from AppGateway AppTier: 10.1.1.0/24 # NSG: Allow app-port from WebTier only DataTier: 10.1.2.0/24 # NSG: Allow SQL/Oracle from AppTier only JumpBox: 10.1.3.0/24 # NSG: Allow 3389 from Bastion only

VNet Peering Configuration

# Hub-to-Spoke peering (Allow gateway transit) $HubToSpoke = @{ Name = "hub-to-spoke-prod" ResourceGroupName = "rg-connectivity" VirtualNetworkName = "vnet-hub-eastus" RemoteVirtualNetworkId = "/subscriptions/.../vnet-spoke-prod" AllowVirtualNetworkAccess = $true AllowForwardedTraffic = $true AllowGatewayTransit = $true # Critical: allows spoke to use hub gateways UseRemoteGateways = $false } Add-AzVirtualNetworkPeering @HubToSpoke # Spoke-to-Hub peering (Use remote gateways) $SpokeToHub = @{ Name = "spoke-prod-to-hub" ResourceGroupName = "rg-prod" VirtualNetworkName = "vnet-spoke-prod" RemoteVirtualNetworkId = "/subscriptions/.../vnet-hub-eastus" AllowVirtualNetworkAccess = $true AllowForwardedTraffic = $false AllowGatewayTransit = $false UseRemoteGateways = $true # Spoke uses hub's ExpressRoute/VPN } Add-AzVirtualNetworkPeering @SpokeToHub

🔗 Phase 2: Hybrid Connectivity Design

2.1 ExpressRoute: The Production Standard

ExpressRoute provides dedicated, private connectivity with predictable latency. For production workloads, this is non-negotiable.

# Step 1: Create ExpressRoute Circuit $ER = @{ Name = "erc-prod-primary" ResourceGroupName = "rg-connectivity" Location = "East US" SkuTier = "Premium" # Required for Global Reach SkuFamily = "MeteredData" # Or "UnlimitedData" for heavy egress ServiceProviderName = "Equinix" PeeringLocation = "Washington DC" BandwidthInMbps = 1000 } New-AzExpressRouteCircuit @ER # Step 2: Configure Private Peering (RFC1918 to Azure) $Peering = @{ Name = "AzurePrivatePeering" Circuit = $ER PeeringType = "AzurePrivatePeering" PeerASN = 65001 # Your on-prem ASN PrimaryPeerAddressPrefix = "10.0.0.0/30" # /30 for each peering SecondaryPeerAddressPrefix = "10.0.0.4/30" VlanId = 100 } Add-AzExpressRouteCircuitPeeringConfig @Peering Set-AzExpressRouteCircuit -ExpressRouteCircuit $ER # Step 3: Connect to Virtual Network Gateway $Connection = @{ Name = "conn-er-to-hub" ResourceGroupName = "rg-connectivity" Location = "East US" VirtualNetworkGateway1 = $HubGateway PeerId = $ER.Id ConnectionType = "ExpressRoute" } New-AzVirtualNetworkGatewayConnection @Connection

2.2 Site-to-Site VPN: Backup & Branch Connectivity

VPN serves as ExpressRoute failover and for branch offices.

# Active-Active VPN Gateway with BGP $LocalGateway = @{ Name = "lgw-onprem-dc" ResourceGroupName = "rg-connectivity" Location = "East US" GatewayIpAddress = "203.0.113.1" # On-prem VPN device public IP AddressPrefix = @("172.16.0.0/12") # On-prem networks BgpAsn = 65001 BgpPeeringAddress = "172.16.255.254" } New-AzLocalNetworkGateway @LocalGateway # VPN Connection with BGP for dynamic routing failover $VPNConnection = @{ Name = "conn-vpn-failover" ResourceGroupName = "rg-connectivity" Location = "East US" VirtualNetworkGateway1 = $VPNGateway LocalNetworkGateway2 = $LocalGateway ConnectionType = "IPsec" RoutingWeight = 10 # Higher than ExpressRoute (0) = backup path EnableBgp = $true SharedKey = "ComplexP@ssw0rd!2026" } New-AzVirtualNetworkGatewayConnection @VPNConnection
ExpressRoute + VPN Coexistence: ExpressRoute carries primary traffic (weight 0), VPN carries backup traffic (weight 10). BGP automatically fails over if ExpressRoute circuit drops.

2.3 SD-WAN Integration (Modern Branch Connectivity)

For distributed offices, integrate SD-WAN with Azure Virtual WAN:

# Azure Virtual WAN for global branch connectivity $vWAN = @{ Name = "vwan-global" ResourceGroupName = "rg-connectivity" Location = "East US" } New-AzVirtualWan @vWAN # Hub in each region $vHub = @{ Name = "vhub-eastus" ResourceGroupName = "rg-connectivity" VirtualWan = $vWAN AddressPrefix = "10.0.128.0/23" # Hub address space Location = "East US" } New-AzVirtualHub @vHub # Connect SD-WAN device (Cisco, VMware, etc.) $VPNSite = @{ Name = "site-branch-nyc" ResourceGroupName = "rg-connectivity" VirtualWan = $vWAN IpAddress = "198.51.100.1" AddressSpace = @("192.168.1.0/24") BgpAsn = 65010 BgpPeeringAddress = "192.168.1.254" } New-AzVpnSite @VPNSite

🛡️ Phase 4: Security & Governance

4.1 Zero Trust Implementation

Identity

# Microsoft Entra ID Conditional Access $CA = @{ Name = "CA-Require-MFA-For-Admins" State = "Enabled" Conditions = @{ Users = @{ IncludeRoles = @("Global Administrator", "Privileged Role Administrator") } Applications = @{ IncludeApplications = @("All") } Locations = @{ IncludeLocations = @("AllTrusted") } } GrantControls = @{ Operator = "OR" BuiltInControls = @("mfa") } } New-AzureADMSConditionalAccessPolicy @CA

Network Segmentation

# Azure Firewall rules for East-West traffic inspection $FW = Get-AzFirewall -Name "afw-hub" -ResourceGroupName "rg-connectivity" # Rule: Allow web tier to app tier only $Rule = @{ Name = "Allow-Web-To-App" SourceAddresses = @("10.1.0.0/24") # WebTier DestinationAddresses = @("10.1.1.0/24") # AppTier DestinationPorts = @("8080") Protocols = @("TCP") Action = "Allow" } $FW.ApplicationRuleCollections.Add($Rule) Set-AzFirewall -AzureFirewall $FW

Micro-segmentation with NSGs

# NSG for DataTier - Deny all inbound except from AppTier $NSG = @{ Name = "nsg-datatier" ResourceGroupName = "rg-prod" Location = "East US" SecurityRules = @( @{ Name = "Allow-SQL-From-App" Priority = 100 Direction = "Inbound" Access = "Allow" Protocol = "Tcp" SourceAddressPrefix = "10.1.1.0/24" SourcePortRange = "*" DestinationAddressPrefix = "10.1.2.0/24" DestinationPortRange = "1433" }, @{ Name = "Deny-All-Inbound" Priority = 4096 Direction = "Inbound" Access = "Deny" Protocol = "*" SourceAddressPrefix = "*" SourcePortRange = "*" DestinationAddressPrefix = "*" DestinationPortRange = "*" } ) } New-AzNetworkSecurityGroup @NSG

4.2 Azure Policy for Governance

# Enforce tagging for cost allocation $Policy = @{ Name = "enforce-cost-tags" DisplayName = "Enforce Cost Center and Environment Tags" Description = "All resources must have CostCenter and Environment tags" Policy = '{ "if": { "not": { "field": "tags[''CostCenter'']", "exists": "true" } }, "then": { "effect": "deny" } }' } New-AzPolicyDefinition @Policy # Assign to subscription $Assignment = @{ Name = "assign-cost-tags" Scope = "/subscriptions/$SubscriptionId" PolicyDefinition = $Policy } New-AzPolicyAssignment @Assignment

✅ Production Readiness Checklist

Before declaring migration complete:

  • Network: ExpressRoute circuits operational with < 5ms latency to hub
  • Security: All NSGs applied, Azure Firewall rules tested, DDoS Standard enabled
  • Identity: Conditional Access policies active, PIM configured for admin roles
  • Monitoring: Log Analytics agents deployed, alerts configured for critical metrics
  • Backup: Azure Backup configured for all VMs; backup tests completed
  • DR: ASR replication healthy; DR drill completed successfully
  • Cost: Budgets set, RI purchased for 1-year baseline, tagging enforced
  • Documentation: Network diagrams updated, runbooks created, on-call rotation established
  • Team: At least 3 engineers certified AZ-104, 1 architect AZ-305
  • Governance: Azure Policy assignments active, RBAC least-privilege verified
